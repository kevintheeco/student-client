## 🔴 버그 / 기술적 문제

### 1. `partial` 채점 시 box가 안 변함 — ✅ 의도된 동작으로 유지
`schedule()` 함수가 `correct` → box+1, `incorrect` → box 1 리셋, `partial`은 box 유지.
사용자가 부분정답 개념을 오늘 바로 계속 복습하길 원하므로 현 동작 유지하기로 결정.
- **위치**: `schedule()` 함수 (line ~492)

### 2. OCR 모델이 항상 Sonnet 4.6 하드코딩
손글씨 제출할 때마다 무조건 Sonnet으로 OCR 사전 체크 실행 (line 1654).
Gemini 모델로 설정한 유저는 Claude key 없으면 터짐. 설정한 모델을 써야 함.
- **위치**: `submit()` 내 OCR check 블록 (line ~1649)
- **수정 방향**: `model:"claude-sonnet-4-6"` 하드코딩 제거, `CFG.model` 또는 `CFG.qmodel` 사용

### 3. API 콜 대부분에 AbortController 없음
`generateFollowup`, `submitFollowup`, `handleDontKnow`, `explainMore`, `handleFollowupHint` 등이
abort signal 없이 실행됨. "← 목록" 눌러 이동하면 요청이 계속 실행되면서 언마운트된
컴포넌트에 setState 시도 → 조용한 에러/메모리 누수.
- **위치**: Study 컴포넌트 내 여러 async 함수들
- **수정 방향**: 각 async 함수에 `abortRef.current?.signal` 전달, 또는 `genId` 가드 추가

### 4. `CFG` 객체가 앱 시작 시 한 번만 로드됨 — ✅ 오인, 버그 아님
재확인 결과 `save()`(KeyForm, line ~717)에서 `CFG.key=key.trim()` 등으로 메모리 값도 갱신함.
API 함수들이 `CFG.model`을 호출 시점에 읽으므로 설정 변경이 새로고침 없이 즉시 반영됨.

### 5. 복습 간격 — ✅ 완료 (시험용으로 단축)
INTERVALS를 `[0,0,1,3,7,16]` → `[0,0,1,2,4,7]`로 변경 (커밋 0433fde). 최대 16일 → 7일.
(원래 메모의 "box1,box2 둘 다 간격 0"은 오류였음 — box2는 INTERVALS[2]=1로 1일이었음.)
- **위치**: `INTERVALS` 상수 (line 490)

## 🟡 데이터 / 안정성 문제

### 6. localStorage에 손글씨 PNG 원본이 쌓임
`getImageBase64()`가 `image/png` 풀 해상도로 내보냄.
세션을 많이 할수록 5MB 한도에 빠르게 도달.
- **수정 방향**: `toDataURL("image/jpeg", 0.7)` 또는 canvas 해상도 제한 (max 800px)

### 7. 백업/내보내기 없음
localStorage가 전부인데 브라우저 데이터 삭제 한 번에 모든 학습 기록 사라짐.
- **수정 방향**: 설정 화면에 "전체 데이터 JSON 내보내기" 버튼 추가

### 8. 개념 수정/추가/삭제 불가
덱 생성 후 개념 목록을 바꿀 방법 없음. AI가 잘못 추출했거나 불필요한 개념이 있어도 계속 나옴.
- **수정 방향**: 덱 카드에 "개념 편집" 버튼 → 개념 목록 편집 화면

## 🟢 UX / 학습 경험 문제

### 9. 세션 종료 시점이 불명확함
오늘 복습할 개념이 없어지면 안내 없이 랜덤 개념이 계속 나옴.
- **수정 방향**: due 개념이 0개일 때 "오늘 복습 다 끝냈어! 🎉" 화면 표시

### 10. 보충질문 완전히 맞게 답해도 항상 `partial` 처리
`submitFollowup`에서 `finalVerdict = r.verdict==="correct" ? "partial" : ...`
스스로 완전히 맞게 보충해도 절대 correct가 못 됨. box도 안 오름.
- **위치**: `submitFollowup()` (line ~1753)
- **수정 방향**: 보충 correct 시 finalVerdict = "partial" 대신 "correct"로 올려서 box 상승 허용

### 11. 홈 덱 카드에 "약함" 숫자 없음
홈 카드에 익힘/대기만 있고 약함(box≤2)은 숨어 있음.
- **수정 방향**: 홈 Stat에 약함 항목 추가 (이미 `deckSummary`가 `weak` 계산 중)

### 12. 개념 하나씩 스킵할 방법 없음
"오늘 이 개념은 패스"가 불가능. 페널티 없는 스킵 버튼 필요.
