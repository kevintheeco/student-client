# 🔐 보안 점검 보고서 (2026-07-04)

> **한 줄 요약: 코드 저장소에 비밀키가 새어나간 것은 없습니다. 다만 AI 중계 서버에 "잠기지 않은 문"이 두 개 있어서, 방치하면 우리(또는 학원)의 AI 사용료가 남의 사용으로 청구될 수 있습니다.**

작성: 개발 (Claude Code 점검 기반) · 대상 독자: 기획자 포함 전체

---

## 먼저, 용어 3개만

| 용어 | 쉬운 설명 |
|------|----------|
| **API 키** | AI 회사(Anthropic, Google 등)가 발급한 **법인카드** 같은 것. 이 키로 AI를 호출하면 키 주인에게 요금이 청구됨 |
| **프록시(중계 서버)** | 사용자와 AI 회사 사이에 있는 **카운터 직원**. 손님 대신 법인카드(키)를 들고 결제해 줌. 키를 손님에게 직접 주지 않기 위해 존재 |
| **환경변수/시크릿** | 키를 코드에 적지 않고 보관하는 **금고**. 코드에는 "금고 3번 칸에 있는 카드를 써라"라고만 적음 |

---

## 발견 사항

### 🔴 1. 백업 중계 서버가 아무나 쓸 수 있는 상태

- **무슨 상황?** 클로드 장애 시 대신 응답해 주는 백업 서버(Gemini/GPT)가 있는데, **출입증 검사 없이** 요청을 받아줍니다. 서버 주소는 앱 코드에 공개되어 있어 누구나 알 수 있습니다.
- **비유하면**: 법인카드가 꽂혀 있는 자판기를 길거리에 내놓은 것. 지나가는 사람 누구나 음료를 뽑을 수 있음.
- **생길 수 있는 일**: 외부인이 우리 서버로 AI를 무료로 쓰고, **요금은 우리에게 청구**됨.
- **개발자 할 일**: 출입증(간단한 인증 토큰) 추가 + 사용량 제한(rate limit) 설정.
- **기획자 할 일**: 없음. 다만 수정 전까지 백업 서버 주소를 외부에 따로 공유하지 말 것.

### 🟠 2. 학원 코드가 사실상 "법인카드 비밀번호"

- **무슨 상황?** 학원용 접속 링크에 붙는 6자리 코드(예: `#academy=○○○○○○`)만 있으면 **그 학원의 클로드 계정으로 무제한 AI 사용**이 가능합니다. 코드가 짧아 추측·유출이 쉽고, 사용량 제한도 없습니다.
- **비유하면**: 쿠폰 코드처럼 생겼는데 실제로는 법인카드 비밀번호. 링크를 아는 사람 = 카드를 쓸 수 있는 사람.
- **생길 수 있는 일**: 링크가 오픈채팅방 등에 퍼지면 학원에 AI 요금 폭탄.
- **개발자 할 일**: 코드를 길고 추측 불가능하게 변경 + 학원별 사용량 상한 + 사용 가능한 AI 모델을 서버에서 고정(현재는 사용자가 비싼 모델을 마음대로 고를 수 있음).
- **기획자 할 일**: **학원 코드가 담긴 링크를 공개 채널(SNS, 오픈카톡, 블로그)에 올리지 않기.** 학원에도 동일하게 안내. 유출이 의심되면 개발자에게 코드 교체 요청.

### 🟡 3. 사용자 API 키가 브라우저에 저장됨

- **무슨 상황?** 현재 앱은 사용자가 자기 API 키를 직접 입력해 쓰는 구조(BYOK)라, 키가 사용자 브라우저 안에 저장됩니다. 악성 스크립트가 끼어들면(XSS 공격) 키를 훔쳐갈 수 있습니다.
- **현재 완화책**: 키는 클라우드 동기화·백업에서 **제외**되도록 이미 잘 처리되어 있음 (이 부분은 잘 만들어졌습니다 👍).
- **개발자 할 일**: 정식 서비스(서버 경유 구조)로 전환할 때 자연히 해소됨. 전환 전까지는 앱에 외부 스크립트를 함부로 추가하지 않기.
- **기획자 할 일**: 새 외부 위젯/스크립트(광고, 분석 도구 등)를 앱에 넣고 싶을 때 개발자와 먼저 상의.

### 🟢 4. 점검 결과 문제 없었던 것들

- 깃 저장소 **전체 이력(커밋 163개)을 스캔 — 비밀키 유출 없음.**
- 앱 코드에 보이는 Firebase 키는 **원래 공개해도 되는 종류**의 키입니다 (진짜 보안은 서버 규칙이 담당하며, 그 규칙은 "본인 데이터만 접근 가능"으로 잘 잠겨 있음).
- 중계 서버의 진짜 키들(Gemini/OpenAI/학원 클로드 키)은 코드가 아닌 **서버 금고(시크릿)**에 보관 중 — 올바른 방식.

## 이번에 새로 생긴 예방 장치

앞으로는 실수로 키가 새는 것을 **자동으로 차단**합니다:

- 개발 도구(Claude Code)가 코드를 쓰거나 커밋할 때마다 **자동 검문**이 돌아서, 비밀키 모양의 값이 코드·커밋에 들어가려 하면 그 자리에서 막습니다.
- 키 값은 코드에 절대 직접 쓰지 않고, "금고 칸 이름(환경변수 이름)"으로만 접근하는 규칙이 강제됩니다.
- 키 값이 들어가는 파일(`.env` 등)은 애초에 깃에 올라가지 않도록 차단 목록에 등록했습니다.
