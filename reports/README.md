# 📋 reports — 기획·개발 공유 보고서

개발자와 기획자가 함께 보는 보고서 폴더입니다. 개발을 몰라도 읽을 수 있게 쓰는 것이 원칙입니다.

## 규칙

- 파일명: `YYYY-MM-DD-주제.md` (예: `2026-07-04-보안점검.md`)
- 전문 용어를 쓸 때는 반드시 쉬운 설명이나 비유를 붙인다.
- 각 발견 사항에는 **"누가, 무엇을 하면 되는지"**를 명시한다 (개발자 할 일 / 기획자 할 일).
- 기획자도 이 폴더에 자유롭게 문서를 추가할 수 있습니다 (예: 기능 제안, 버그 제보).

## 심각도 표기

| 표시 | 의미 |
|------|------|
| 🔴 높음 | 지금 당장 문제가 생길 수 있음. 빠른 조치 필요 |
| 🟠 중간 | 당장은 아니지만 방치하면 위험 |
| 🟡 낮음 | 알고는 있어야 함. 계획된 개선에 포함 |
| 🟢 양호 | 점검 결과 문제 없음 |
