- 브랜치명이 주어지면 (기획자 바이브코딩 브랜치 인수 리뷰): `git fetch` 후 `git diff main...{branch}` 기준으로 리뷰한다.

- `/CLAUDE.md`
- `/docs/ARCHITECTURE.md`
- `/docs/ADR.md`

## 체크리스트

1. **아키텍처 준수**: ARCHITECTURE.md에 정의된 디렉토리 구조를 따르고 있는가?
2. **기술 스택 준수**: ADR에 정의된 기술 선택을 벗어나지 않았는가?
3. **테스트 존재**: 새로운 기능에 대한 테스트가 작성되어 있는가?
4. **CRITICAL 규칙**: CLAUDE.md의 CRITICAL 규칙을 위반하지 않았는가?
5. **빌드 가능**: 빌드 명령어가 에러 없이 통과하는가?
6. **보안**: API 키/시크릿이 클라이언트 코드나 커밋에 노출되지 않았는가? (기획자 브랜치 리뷰 시 특히 확인)
