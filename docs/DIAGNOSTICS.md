# 진단 인텔리전스 레이어 (v1)

> "이 문제를 틀린 이유는 어떤 개념의 결여에 기인한 것인가"를
> 시간적(시계열)·서술적으로 설명하는 수학 진단 엔진.
> 손글씨 기반 끈질긴 과외 선생님(니가교수)의 채점 데이터를
> 지식 그래프 + 능력 요인 모델로 가공한다.

## 이론적 근거

| 구성 요소 | 근거 문헌 |
|---|---|
| 능력 요인 5가지 | Kilpatrick, Swafford & Findell, **Adding It Up** (National Research Council, 2001) — 수학적 숙련도의 5 strand: 개념적 이해·절차적 유창성·전략적 역량·적응적 추론·생산적 성향 |
| 선수관계 그래프·역추적 | **Knowledge Space Theory** (Doignon & Falmagne, 1985) — ALEKS의 기반 이론. 지식 상태는 선수관계로 닫힌 부분집합 |
| 능력 추정(측정 엔진 v2) | **Rasch/1PL IRT** — PISA·TIMSS가 쓰는 국제 표준 척도. 스트리밍 추정은 **Elo** (Klinkenberg, Straatemeier & van der Maas 2011, Math Garden — Elo는 Rasch의 온라인 확률적 경사 추정량) |
| 망각 모델 | **Half-Life Regression** (Settles & Meeder 2016, Duolingo)의 근사 — 반감기가 연습량에 따라 성장(간격 효과) |
| 적응형 난이도 | **CAT 최대 정보 원리** — 표적 성공률 70~80%가 학습·측정 효율 최대 지점 (Math Garden의 적응 규칙) |
| 모델 자기 검증 | **Brier score·신뢰도 다이어그램** — 확률 예측 모델 평가의 표준 (Brier 1950) |
| 숙련도 정교화(로드맵) | **Bayesian Knowledge Tracing** (Corbett & Anderson, 1994), 2PL/다차원 IRT — 학원 데이터로 문항 실측 보정 후 교체 |

## 구조

```
채점 이벤트(학습·후속질문·퀴즈·시험)
  → attempts.js  시도 로그 (ng:attempts, append-only, 클라우드 동기화 포함)
      · AI 채점에 FACTORS 평가 추가 (개념/계산/전략/추론 0~2)
      · 개념·단원 텍스트 → 그래프 노드 매핑 (knowledgeGraph.matchNode)
  → mastery.js   노드별 숙련도(EWMA α=0.3) · 요인 주간 시계열 · 최소제곱 회귀 기울기
      · 생산적 성향(끈기)은 행동 지표: 오답 후 재도전율 + 소크라테스 후속질문 참여율
  → knowledgeGraph.js  중1~미적분 71개 단원 노드 + 선수관계 가중 엣지 (~90개)
      · traceRootCauses: 의심도 = 경로 가중치(엣지 곱) × (1 − 선수 숙련도)
      · impactOf: 이 단원이 무너지면 영향받는 후속 단원 수(전이적)
  → diagnosis.js  AI 서술식 진단 (근본 원인 + 인과 서사 + 보강 순서, ng:diag:* 캐시)
  → views/Insight.jsx  대시보드: 지식 그래프 / 5요인 레이더 / 성장 회귀
```

### 피드포워드 루프 — 끈질긴 과외
오개념 라벨은 노드별 사전(`ng:misclex`)에 누적되고, 같은 라벨이 2회 이상
반복되면 '이 학생이 반복해서 밟는 함정'으로 승격된다. Study의 모든 출제 경로
(일반·기출 변형·오답 후 변형)는 이 함정 목록을 프롬프트에 주입받아
**그 실수를 하면 틀리게 되는 문제를 정조준**해서 낸다 — 진단이 곧 처방이 되는 루프.
시험 리포트(Exam)에는 오류 성격 분석(실수 vs 개념 결여 분포 + 처방 문장 +
요인별 증거 막대)이 들어가 학부모 상담 자료가 "점수"가 아니라 "오답의 성격"을 말한다.

### 측정 엔진 v2 (core/rasch.js) — 세계 표준 측정 모델로 가는 뼈대

시도 로그를 시간순으로 **순수 재생(replay)** 하며 노드별 능력 θ(로짓)를 추정한다.
저장 상태가 없어 어떤 시점의 능력도 로그만으로 재현 가능 — 감사 가능한 측정.

1. **능력-난이도 분리**: P(정답)=σ(θ−b). 난이도 b는 box(SRS 단계)·문항 유형·출처의
   프록시 — 학원 데이터가 모이면 문항별 실측 난이도(2PL)로 교체되는 자리.
2. **Elo 갱신**: θ += K(n)·(s−P). 학습률 K는 관측이 쌓이면 감소(초반 민첩, 이후 안정).
3. **망각 감쇠**: 관측이 없으면 θ가 사전값으로 수축. 반감기 = 30일+12일×n (상한 240일)
   — 연습할수록 오래간다. "3달 전의 숙련도"가 지금의 숙련도로 표시되는 왜곡 제거.
4. **불확실성**: 누적 피셔 정보로 표준오차 → 노드 상세에 "74% ±9p" 신뢰구간 표기.
   측정은 점이 아니라 구간이다.
5. **그래프 증거 전파**: 후속 단원의 성공은 선수 단원 사용의 증거 — 선수 엣지(w≥0.5)로
   갱신의 25%를 전파해, 직접 안 풀어본 단원도 '간접 추정'(점선 링)으로 표시.
   지식 그래프와 심리측정의 결합 — 우리만의 차별점.
6. **자기 검증**: 매 관측 전의 P가 곧 모델의 예측. Brier score와 예측 구간별
   신뢰도(예측 vs 실제)를 대시보드에 공개 — 모델 개선의 기준선이자,
   "우리 측정이 얼마나 정확한가"를 학원·학부모에게 증명하는 수단.
7. **적응형 출제 보정**: 추정 성공률을 출제 프롬프트에 주입, 표적 성공률 70~80%로
   난이도를 조정 — 측정과 학습 효율을 동시에 최대화하는 지점.

### 설계 원칙 (v1)
- **해석 가능성 우선**: EWMA·선형회귀·명시적 엣지 가중치 등 투명한 모델만 사용.
  블랙박스 대신 모든 수치에 "왜 이 값인가"를 보여줄 수 있다.
- **엣지 가중치 = 사전(prior)**: 현재 값은 교육과정 전문성으로 수기 설정한 초안.
  학원 데이터가 결합되면 학습 가능한 파라미터가 된다(아래 로드맵).
- **텍스트만 축적**: 손글씨 이미지는 로그에 넣지 않아 가볍고(2,000건 ≈ 400KB)
  Firestore 동기화 대상에 포함된다 → 기기·학원 어디서든 같은 성장 데이터.

## 데이터 사전 — 시도 레코드 v2

**무엇을 기록할 것인가의 원칙**: 수천 명 규모에서 살아남는 데이터는
"풀이 원문"이 아니라 "부호화된 신호"다. 자유 텍스트는 집계가 안 되고,
이미지는 저장이 안 터지는 규모가 없다. 고정 enum + 수치 + 길이 상한 텍스트만 남긴다.

| 필드 | 형식 | 왜 기록하나 |
|---|---|---|
| `t, src` | 시각, `study/followup/quiz/exam/dontknow` | 시계열·맥락 |
| `nodeId, concept, unit` | 그래프 노드 + 텍스트(≤60자) | 지식 그래프 연결 |
| `verdict, score/points` | enum, 숫자 | 기본 정오·부분점수 |
| `err` | **`slip/concept/strategy/interpret/notation/blank`** | **가장 가치 있는 1비트: 실수 vs 개념 결여.** 처방이 정반대(검산 훈련 vs 선수 개념 보강). BKT의 slip 파라미터, VanLehn 오류 진단 연구의 표준 구분 |
| `stage` | `setup/compute/interpret` | 첫 오류가 난 풀이 단계 — 식 세우기에서 무너지는지 계산에서 무너지는지 |
| `misc` | 라벨 ≤48자 | 오개념 패턴("부호 분배 실수") — 향후 학생 간 클러스터링으로 통제어휘 승격 |
| `gap, gapType` | 텍스트 ≤140자, enum | 서술식 진단의 증거 문장 |
| `factors` | `{cu,pf,sc,ar}` 0~1 | NRC 5-strand 중 AI 평가 4개 |
| `dur` | 초 | 풀이 시간 — IRT의 표준 공변량(속도-정확도). 같은 정답도 20초와 300초는 다른 숙련도 |
| `ink {st,pg}` | 획수·페이지 수 | 이미지 없이 풀이 분량을 수치화 |
| `hint, ocr` | 횟수 | 힌트 요청('한계야')·필기 재작성 — 자립도·명료성 신호 |

**이중 저장 구조**: 원시 로그(`ng:attempts`)는 최근 2,000건 순환(~400KB 상한),
노드별 평생 집계(`ng:attagg`: n·점수합·오류 유형 분포·풀이 시간 누계)는 영구 —
로그가 밀려나도 학생의 이력은 잃지 않는다. `slip` 오답은 숙련도 하한 0.6으로
반영해(개념은 아는 상태) 실수 때문에 그래프가 빨개지는 왜곡을 막는다.

**기록하지 않는 것**: 손글씨 이미지(채점 후 폐기 — 용량·프라이버시),
문제 전문(덱에 이미 있음), AI 해설 전문(재생성 가능). 원칙: 재생성 가능한 것은
저장하지 않고, 재현 불가능한 학생의 행동·상태 신호만 남긴다.

## 학원 빅데이터 결합 로드맵

1. **엣지 가중치 학습**: 학생 N명의 시도 로그로 로지스틱 회귀 —
   "선수 노드 숙련도들 → 후속 노드 정답 확률"의 계수가 곧 엣지 가중치.
   수기 prior를 데이터로 보정.
2. **요인 → 성적 회귀**: 5요인 점수로 시험 점수를 설명하는 회귀모델 →
   "이 학생의 점수를 올리는 최단 경로는 어떤 요인인가"를 정량화.
   (대수 실력과 기하 실력의 상관, 계산력의 매개 효과 등 검증 가능)
3. **BKT/IRT 교체**: 노드별 EWMA → BKT(guess/slip 파라미터)로,
   문항 난이도는 IRT로 보정. mastery.js의 인터페이스는 유지.
4. **다중 학생 (구현됨)**: 학원 기기는 학생 명단(ng:aca:students)에서 활성 학생을
   선택 — 모든 시도 레코드에 sid가 찍히고 평생 집계·오개념 사전·진단 캐시가
   학생별 키로 분리된다. 학생별 성장 인사이트를 학원 화면에서 바로 연다.
   서버가 생기면 sid가 그대로 서버 학생 계정 키가 된다.
5. **학원 리포트**: 시험 리포트(Exam.jsx 학부모 상담 리포트)에
   요인 성장 그래프·근본 원인 서사를 통합 — "점수"가 아니라
   "개별 factor가 얼마나 늘었는가"를 보여주는 상담 자료.

## 차별점 (vs 단순 변형문제 재출제형 서비스)

- 오답 → 같은 단원 변형문제가 아니라, **그래프 역추적으로 뿌리 단원부터 보강**.
- 손글씨 풀이 전체를 AI가 읽으므로 "답은 맞았지만 핵심 개념이 풀이에서 누락"
  같은 신호까지 요인 점수에 반영된다.
- 끈기(생산적 성향)를 행동 데이터로 측정 — 학습 태도의 성장도 리포트된다.
