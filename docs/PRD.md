# PRD: 니가 교수 (YouareProfessor)

## 목표
학생이 스스로 설명하게 만들고, 그 과정을 분석해 약점을 채우는 자기주도 학습 플랫폼.
"설명할 수 있어야 진짜 아는 것이다" — 지식을 소비하는 것이 아니라 설명하는 능력을 기른다.

## 배경
경제학과 3학년 창업자가 선형대수·수리통계 같은 과목 앞에서 무너지는 경험을 하며,
"이렇게 어려움을 겪는 게 나 혼자만은 아닐 거라고" 깨달아 2026년 6월 13일 개발 시작.
기말고사를 준비하며 앱을 병행 개발했고, 직접 써서 성적이 올랐다.
첫 번째 수요자는 '나 자신' — 자신처럼 어려움을 겪는 학생들을 위해 만든다.

기존 AI 교육의 한계: 대부분은 "AI가 학생에게" 일방 전달. 의지와 습관이 약한 학생은 떠먹여줘도 받아먹지 못한다.
우리는 방향을 거꾸로, 학생이 직접 설명하게 한다.

## 사용자
- **B2C**: 스스로 공부하는 중·고등학생, 대학생 (특히 혼자 공부가 버거운 학생)
- **B2B**: 학원·교육기관 (AI 진단·레벨테스트를 사람 추가 없이 1:1 수준으로 제공)
- **B2G**: 정부·지자체 교육복지 (서울런 모델, 교육 격차를 좁히는 계층 사다리)

## 핵심 기능
1. **개념 덱 생성** — PDF·이미지·손글씨 자료 업로드 → AI가 핵심 개념 자동 추출
2. **스페이스드 리피티션 학습 루프** — box 0~5 레벨, 복습 간격 자동 조절 (최대 7일)
3. **손글씨 답안 → AI 채점 + 피드백** — 정답이어도 풀이 과정 속 구멍을 찾아냄
4. **변형 문제로 굳히기** — 틀렸을 때 같은 개념의 변형 문제를 새로 출제해 재도전
5. **튜터 모드** — 소크라테스식 질문으로 개념 깊이 탐구
6. **시험 모드** — 학원·교육기관용 일괄 출제·채점 (B2B)
7. **오답노트** — 약한 개념(box≤2) 모아보기 + 집중 복습

## 유저 플로우

### B2C 개인 학습
```
온보딩 → API 키 설정 (Claude or Gemini)
  ↓
홈 화면 → 과목 탭 → 덱 목록
  ↓
새 덱 만들기 → 자료 업로드 (PDF/이미지/손글씨)
  → OCR + 개념 추출 중... → 덱 생성 완료
  ↓
학습 시작
  → 오늘 복습할 개념 선택 (SRS)
  → 문제 표시
  → 손글씨로 답변 작성
  → AI 채점 결과 확인 (correct/partial/incorrect)
    → partial/incorrect: 보충질문 or 변형 문제 선택
  → 다음 개념 or 세션 종료
  ↓
오답노트 → 약한 개념 집중 복습
```

### B2B 학원 모드
```
URL: {도메인}/#academy={학원코드}
  ↓
학원 전용 UI (academy-skin) 진입
  → 학원이 미리 세팅한 덱/과목 선택
  → 시험 모드 (Exam): 문제 일괄 출제 → 학생 답변 → 일괄 채점
  → 결과 리포트 (정답률, 약점 개념 목록)
```

## MVP 제외 사항
- 소셜 기능 (랭킹, 친구, 공유)
- 자체 콘텐츠 마켓플레이스 (교재 판매)
- 앱스토어 네이티브 앱 (웹앱으로 운영)
- 학습 분석 어드민 대시보드
- 실시간 다중 사용자 세션

- 공통: 모바일 우선, 손글씨 입력 최적화 (태블릿/S펜 고려)
